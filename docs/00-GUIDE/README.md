# 🧭 00-GUIDE — Guía principal del proyecto

> Esta carpeta contiene la guía esencial que **todo agente (humano o IA) debe leer primero** antes de trabajar en Vanchi.

## ¿Qué contiene?

| Archivo | Propósito | Cuándo leerlo |
|---------|-----------|---------------|
| [`CONTEXT.md`](./CONTEXT.md) | **⬅️ LEER PRIMERO** — Stack, estado actual, estructura del proyecto | Siempre, al empezar |
| [`NAVIGATION.md`](./NAVIGATION.md) | GPS: qué documentos leer según la tarea | Antes de cada tarea |
| [`WORKFLOW.md`](./WORKFLOW.md) | Ciclo Spec → Code → Verify | Antes de implementar |
| [`STACK-PATTERNS.md`](./STACK-PATTERNS.md) | Reglas obligatorias del stack técnico | Al escribir código |
| [`LECCIONES-APRENDIDAS/`](./LECCIONES-APRENDIDAS/README.md) | Decisiones clave (1 archivo = 1 lección) | Cuando necesites contexto histórico |

## Orden de lectura

1. `CONTEXT.md` — entender el proyecto
2. `NAVIGATION.md` — saber qué más leer según tu tarea
3. `WORKFLOW.md` — entender cómo trabajamos
4. `STACK-PATTERNS.md` — antes de escribir código
5. `LECCIONES-APRENDIDAS/` — cuando quieras evitar errores pasados
